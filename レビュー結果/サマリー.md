## 横断的な共通指摘事項

以下は**全設計書に共通して見られる指摘事項**です。優先して対応することを推奨します。

| 優先度 | 観点 | 内容 |
|--------|------|------|
| 🔴 高 | 変更履歴管理 | ほぼ全設計書で変更履歴テーブル（変更日・変更者・変更内容）が欠如 |
| 🔴 高 | エラーハンドリング | APIエラーレスポンス（4xx/5xx）・エラーケース網羅・エラー遷移先が未定義 |
| 🔴 高 | セキュリティ | XSS・CSRF・SQLインジェクション対策の明記不足 |
| 🔴 高 | 認証・認可 | 対象ユーザー定義・ログイン必須要件・未認証時の挙動が未記載 |
| 🟡 中 | ログ出力仕様 | ログレベル・出力項目・出力先・ローテーション仕様が未定義 |
| 🟡 中 | タイムアウト・リトライ | DB・外部API・SMTP のタイムアウト値とリトライ方針が未定義 |
| 🟡 中 | レスポンシブ対応 | 画面設計書全般で対応要否・ブレイクポイントが未記載 |
| 🟡 中 | ブラウザバック動作 | 二重送信・二重注文リスクに対する動作定義が欠如 |
