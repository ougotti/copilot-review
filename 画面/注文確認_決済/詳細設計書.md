---

## 1. 処理フロー

```
[ページ表示]
    │
    ├─ 認証チェック: セッションCookie必須（未ログイン時はSCR-LOGINへ）
    │
    ├─ カート情報を取得して読み取り専用表示
    │       カートが空の場合はSCR-003へリダイレクト
    │
    └─ 登録済み配送先・決済方法をフォームに自動入力

[注文確定ボタン押下]
    │
    ├─ CSRFトークン検証
    │
    ├─ 認可チェック: 操作ユーザーのセッションのカートのみ操作可能
    │
    ├─ クライアントサイド: 「注文を確定する」ボタンを押下後即時にロック（二重送信防止）
    │
    ├─ サーバーサイドバリデーション（全入力項目）
    │       エラーあり → 400レスポンス、同画面にエラー表示
    │
    ├─ [トランザクション開始]
    │       ├─ 在庫チェック（SELECT FOR UPDATE）
    │       │       不足あり → rollback → 409エラー表示
    │       ├─ orders テーブル INSERT
    │       ├─ order_items テーブル INSERT（カートアイテム分）
    │       ├─ products.stock を減算 UPDATE
    │       └─ cart_items DELETE
    ├─ [トランザクションコミット（決済前）]
    │
    ├─ 決済APIコール（外部決済代行）
    │       失敗 → orders.payment_status を 'failed' に更新、決済エラー表示
    │       警告: 決済レスポンス受信前にタイムアウトした場合、二重請求リスクあり（無リトライ）
    │
    ├─ orders.payment_status を 'paid' に UPDATE
    │
    └─ 注文完了画面へリダイレクト（order_id付き、PRGパターン）
```

## 2. API定義

### POST /api/orders

**認証**: セッションCookie必須

**CSRFトークン**: リクエストヘッダーに `X-CSRF-Token` を含めること

**リクエストフィールド**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| shipping.name | string | ○ | 氏名（50文字以内） |
| shipping.postal_code | string | ○ | 7桁数字 |
| shipping.prefecture | string | ○ | 都道府県 |
| shipping.address | string | ○ | 住所（200文字以内） |
| shipping.phone | string | ○ | 10〜11桁数字 |
| payment.method | string | ○ | `credit_card` / `convenience` |
| payment.token | string | △ | カードトークン（credit_card時必須） |

**リクエストボディ**

```json
{
  "shipping": {
    "name": "山田 太郎",
    "postal_code": "1000001",
    "prefecture": "東京都",
    "address": "千代田区千代田1-1",
    "phone": "0312345678"
  },
  "payment": {
    "method": "credit_card",
    "token": "tok_xxxxxxxxxxxxxxxx"
  }
}
```

**レスポンス (201 Created)**

```json
{
  "order_id": 10001,
  "message": "注文が完了しました"
}
```

**エラーレスポンス**

| ステータス | エラーコード | 説明 |
|-----------|-------------|------|
| 400 | `VALIDATION_ERROR` | 入力バリデーションエラー |
| 402 | `PAYMENT_FAILED` | 決済失敗 |
| 409 | `OUT_OF_STOCK` | 在庫不足 |
| 500 | `INTERNAL_SERVER_ERROR` | DBエラー |

```json
// 400
{ "error": "VALIDATION_ERROR", "message": "入力内容に誤りがあります" }
// 402
{ "error": "PAYMENT_FAILED", "message": "決済に失敗しました。カード情報をご確認ください" }
// 409
{ "error": "OUT_OF_STOCK", "message": "在庫が不足しています。カートに戻ってご確認ください" }
// 500
{ "error": "INTERNAL_SERVER_ERROR", "message": "システムエラーが発生しました" }
```

## 4. 在庫減算 SQL（トランザクション内）

```sql
UPDATE products
SET stock = stock - :quantity
WHERE id = :product_id AND stock >= :quantity;
-- 更新件数 = 0 の場合は在庫不足としてロールバック
```

## 5. 外部決済連携

| 項目 | 内容 |
|------|------|
| 決済代行 | Stripe（想定） |
| 認証方式 | APIキー（サーバーサイド保持） |
| カード情報 | フロント側でトークン化し、rawカード番号をサーバーに送信しない |
| タイムアウト | 30秒 |
| リトライ | なし（ユーザーに再試行を促す） |

## 6. エラーハンドリング

| エラー | 対応 | ログレベル | ログ内容 |
|--------|------|-----------|----------|
| バリデーションエラー | 400レスポンス、同画面にエラー表示 | INFO | 詳細エラー内容 |
| 在庫不足 | 409レスポンス、エラーメッセージ表示 | INFO | product_id・要求数・在庫数 |
| 決済失敗 | 402レスポンス、エラーメッセージ表示 | WARN | 決済エラーコード・注文ID |
| 決済タイムアウト（30秒） | 警告ログ出力、管理チームに通知 | ERROR | 注文ID・タイムアウト発生時刻 |
| CSRFトークン不一致 | 403レスポンス | WARN | リクエストIP・セッションID |
| DBエラー | トランザクションロールバック、500エラー画面（SCR-ERR） | ERROR | リクエストID・セッションID・SQLエラーコード |

**内部情報の非開示**: 500エラー画面にスタックトレース・DB内部情報を含めない。

**タイムアウト**: DBクエリタイムアウト5秒、決済APIタイムアウト30秒。
