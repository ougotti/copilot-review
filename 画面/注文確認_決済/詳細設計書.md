## 1. 処理フロー

```
[ページ表示]
    │
    └─ カート情報を取得して読み取り専用表示

[注文確定ボタン押下]
    │
    ├─ サーバーサイドバリデーション（全入力項目）
    │
    ├─ 在庫チェック（全カートアイテム）
    │       不足あり → ロールバック、エラー表示
    │
    ├─ トランザクション開始
    │       ├─ orders テーブル INSERT
    │       ├─ order_items テーブル INSERT（カートアイテム分）
    │       ├─ products.stock を減算 UPDATE
    │       └─ cart_items DELETE
    │
    ├─ 決済APIコール（外部決済代行）
    │       失敗 → トランザクションロールバック、エラー表示
    │
    ├─ orders.payment_status を「決済済」に UPDATE
    │
    ├─ トランザクションコミット
    │
    └─ 注文完了画面へリダイレクト（order_id付き）
```

## 4. 在庫減算 SQL（トランザクション内）

```sql
UPDATE products
SET stock = stock - :quantity
WHERE id = :product_id AND stock >= :quantity;
-- 更新件数 = 0 の場合は在庫不足としてロールバック
```

## 5. 外部決済連携

| 項目 | 内容 |
|------|------|
| 決済代行 | Stripe（想定） |
| 認証方式 | APIキー（サーバーサイド保持） |
| カード情報 | フロント側でトークン化し、rawカード番号をサーバーに送信しない |
| タイムアウト | 30秒 |
| リトライ | なし（ユーザーに再試行を促す） |

## 6. エラーハンドリング

| エラー | 対応 |
|--------|------|
| バリデーションエラー | 各入力フィールドにエラーメッセージ表示 |
| 在庫不足 | エラーメッセージ表示、カートへ戻るリンク提示 |
| 決済失敗 | エラーメッセージ「決済に失敗しました。カード情報をご確認ください。」 |
| タイムアウト | エラーメッセージ「通信エラーが発生しました。再度お試しください。」 |
| DB エラー | トランザクションロールバック、500エラー画面、ログ出力 |
