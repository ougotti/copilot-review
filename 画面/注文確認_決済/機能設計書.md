# 機能設計書 – 注文確認・決済画面

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-004 |
| 画面名 | 注文確認・決済 |
| 作成日 | 2026-04-08 |
| バージョン | 1.0 |

---

## 1. 概要

注文内容の最終確認と配送先情報の入力、決済手段の選択を行う画面。
「注文を確定する」ボタン押下により注文が確定し、注文完了画面へ遷移する。

---

---

## 3. 機能一覧

| No | 機能名 | 説明 |
|----|--------|------|
| F-01 | 注文内容確認 | カート内商品・合計金額を読み取り専用で表示する |
| F-02 | お届け先入力 | 配送先の氏名・住所等を入力する |
| F-03 | 決済方法選択 | クレジットカードまたはコンビニ払いを選択する |
| F-04 | クレジットカード入力 | カード番号・有効期限・セキュリティコードを入力する |
| F-05 | 注文確定 | 入力内容を検証し、注文レコードを生成して決済を実行する |

---

## 4. 入力項目

| 項目名 | 種別 | 必須 | バリデーション |
|--------|------|------|----------------|
| 氏名 | テキスト | ○ | 最大50文字 |
| 郵便番号 | テキスト | ○ | 7桁数字（ハイフン自動除去） |
| 都道府県 | セレクト | ○ | |
| 市区町村・番地 | テキスト | ○ | 最大200文字 |
| 電話番号 | テキスト | ○ | 10〜11桁数字 |
| 決済方法 | ラジオ | ○ | |
| カード番号 | テキスト | △ | 16桁数字（カード選択時必須） |
| 有効期限 | テキスト | △ | MM/YY形式（カード選択時必須） |
| セキュリティコード | テキスト | △ | 3〜4桁（カード選択時必須） |

---

## 5. 画面遷移

| 操作 | 遷移先 |
|------|--------|
| 「注文を確定する」成功 | 注文完了画面（SCR-005） |
| 「カートに戻る」リンク | カート画面（SCR-003） |

---

## 6. 業務ルール

- 注文確定時に在庫の最終チェックを行い、在庫不足の場合はエラー表示しカートへ誘導する
- クレジットカード情報はサーバーに保存しない（決済代行サービスへトークン化して送信）
- コンビニ払い選択時は注文後に支払い番号を発行し注文完了画面に表示する
