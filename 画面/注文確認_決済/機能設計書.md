## 1. 概要

注文内容の最終確認と配送先情報の入力、決済手段の選択を行う画面。
「注文を確定する」ボタン押下により注文が確定し、注文完了画面へ遷移する。

**対象ユーザー**: ログイン必須。未ログインユーザーがアクセスした場合はログイン画面（SCR-LOGIN）へリダイレクトする。

## 3. 機能一覧

| No | 機能名 | 説明 |
|----|--------|------|
| F-01 | 注文内容確認 | カート内商品・合計金額を読み取り専用で表示する |
| F-02 | お届け先入力 | 配送先の氏名・住所等を入力する |
| F-03 | 決済方法選択 | クレジットカードまたはコンビニ払いを選択する |
| F-04 | クレジットカード入力 | カード番号・有効期限・セキュリティコードを入力する |
| F-05 | 注文確定 | 入力内容を検証し、注文レコードを生成して決済を実行する |

## 4. 入力項目

| 項目名 | 種別 | 必須 | バリデーション | 初期値 |
|--------|------|------|----------------|--------|
| 氏名 | テキスト | ○ | 最大50文字 | 登録済み氏名（未登録時は空白） |
| 郵便番号 | テキスト | ○ | 7桁数字（ハイフン自動除去） | 登録済み郵便番号 |
| 都道府県 | セレクト | ○ | | 登録済み都道府県 |
| 市区町村・番地 | テキスト | ○ | 最大200文字 | 登録済み住所 |
| 電話番号 | テキスト | ○ | 10～11桁数字 | 登録済み電話番号 |
| 決済方法 | ラジオ | ○ | | 前回利用履歴から自動設定（初回はクレジットカード） |
| カード番号 | テキスト | △ | 16桁数字（カード選択時必須） | |
| 有効期限 | テキスト | △ | MM/YY形式（カード選択時必須） | |
| セキュリティコード | テキスト | △ | 3～4桁（カード選択時必須） | |

## 5. 画面遷移

| 操作 | 遷移先 |
|------|--------|
| 「注文を確定する」成功 | 注文完了画面（SCR-005） |
| バリデーションエラー | 同画面にエラーメッセージ表示（ページ遷移なし） |
| 「カートに戻る」リンク | カート画面（SCR-003） |
| 決済失敗 | 同画面に決済エラーメッセージ表示 |
| システムエラー（500系） | エラー画面（SCR-ERR） |
| 未ログインアクセス | ログイン画面（SCR-LOGIN） |
| カートが空の状態でアクセス | カート画面（SCR-003）へリダイレクト |
| 注文確定後にブラウザバック | PRGパターンにより注文完了画面に留まる（二重注文防止） |

## 6. 業務ルール

- 注文確定時に在庫の最終チェックを行い、在庫不足の場合はエラー表示しカートへ誘導する
- クレジットカード情報はサーバーに保存しない（決済代行サービスへトークン化して送信）
- コンビニ払い選択時は注文後に支払い番号を発行し注文完了画面に表示する- コンビニ払い選択時の上限金額は、8万円とする
- 二重注文防止: 「注文を確定する」ボタンは押下後即時にロック（クライアントサイド）。PRGパターンでブラウザバックによる再送信を防止する
- 注文アイテム数の上限は100点（超過時は注文不可としエラー表示）
- 注文キャンセルは注文完了画面からは不可能（キャンセルは別途マイページ又はカスタマーサポートへ連絡）
