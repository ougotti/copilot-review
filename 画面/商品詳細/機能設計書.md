## 1. 概要

特定の商品の詳細情報（画像・説明・価格・在庫）を表示し、カートへの追加を行う画面。
商品一覧（SCR-001）からの遷移が主なエントリーポイント。外部検索エンジン経由のダイレクトアクセスも想定する。

**対象ユーザー**: ゲスト（未ログイン）・会員（ログイン済み）どちらもアクセス可能。
お気に入り登録はログイン済みユーザーのみ利用可能。

## 2. 画面レイアウト

```
┌─────────────────────────────────────────┐
│  ヘッダー（ロゴ / カートアイコン）      │
├─────────────────────────────────────────┤
│  パンくずリスト: トップ > 商品一覧 > 商品名 │
├──────────────────┬──────────────────────┤
│  商品画像（メイン）│  商品名              │
│                  │  価格（税込表示）     │
│  サムネイル一覧  │  在庫状況            │
│                  │  数量セレクタ（初期値:1）│
│                  │  [カートに追加]      │
│                  │  [お気に入り登録]    │
├──────────────────┴──────────────────────┤
│  商品説明（リッチテキスト）              │
├─────────────────────────────────────────┤
│  関連商品一覧（0件時: セクション非表示）  │

```

**レスポンシブ対応**: スマートフォン・タブレット対応必須。モバイルでは縦並びレイアウトに切り替える。

## 3. 機能一覧

| No | 機能名 | 説明 |
|----|--------|------|
| F-01 | 商品情報表示 | 商品ID に対応する詳細情報を表示する |
| F-02 | 画像ギャラリー | メイン画像とサムネイルをスライダーで表示する。画像未登録時はプレースホルダー画像を表示する |
| F-03 | 数量選択 | 購入数量を1〜在庫数の範囲で選択する（初期値: 1） |
| F-04 | カートへ追加 | 指定数量をカートに追加する。カートに既存商品あり時は数量を加算する |
| F-05 | お気に入り登録 | ログイン済みユーザーがお気に入りに登録できる。未ログイン時はログイン画面（SCR-LOGIN）へ誘導 |
| F-06 | 関連商品表示 | 同一カテゴリの商品を最大4件表示する（0件時はセクション非表示） |
| F-07 | パンくずリスト | トップ > 商品一覧（SCR-001） > 商品名 のパンくずリストを表示する |

---

## 4. 入力項目

| 項目名 | 種別 | 必須 | バリデーション | 初期値 |
|--------|------|------|----------------|--------|
| 数量 | セレクトボックス | ○ | 数値のみ。最小値: 1、最大値: min(9, 在庫数) | 1 |

## 5. 画面遷移

| 操作 | 遷移先 |
|------|--------|
| 「カートに追加」押下 | カート画面（SCR-003） |
| 関連商品クリック | 商品詳細画面（SCR-002）※別商品 |
| パンくずリスト | 商品一覧画面（SCR-001） |
| お気に入りボタン押下（未ログイン） | ログイン画面（SCR-LOGIN） |
| システムエラー発生（500系） | エラー画面（SCR-ERR） |

---

## 6. 業務ルール

- 在庫数が0の場合、数量セレクタとカート追加ボタンを非表示にし「在庫なし」表示
- 数量は最大9個まで選択可（在庫数が9未満の場合は在庫数が上限）
- 未ログインユーザーがお気に入りボタンを押した場合、ログイン画面（SCR-LOGIN）へ遷移
- 非公開商品・販売期間外商品・削除済み商品のURLへ直接アクセスした場合、404ページを表示
- カートに既に同一商品が入っている場合は数量を加算する（別エントリーは作成しない）
- 価格は税込表示（消費税法に基づく総額表示義務。税率10%を適用）
- 数量上限9個は業務要件による制限
- ブラウザバック時: カート追加後にブラウザバックしても再度カート追加ボタンを押せる状態とする（二重送信はサーバー側で数量加算として処理）
