## 1. 処理フロー

```
[ページ表示]
    │
    ├─ リクエストパラメータ取得（category, keyword, sort, page）
    │
    ├─ 入力バリデーション
    │       keyword: 最大100文字、SQLインジェクション対策エスケープ
    │       page: 数値、最小1
    │
    ├─ 商品マスタ検索（SELECT）
    │       WHERE 公開フラグ = 1
    │         AND（category指定時）カテゴリID = :category
    │         AND（keyword指定時）商品名 LIKE '%:keyword%'
    │              OR 商品説明 LIKE '%:keyword%'
    │       ORDER BY [並び替え条件]
    │       LIMIT 20 OFFSET (page-1)*20
    │
    └─ レスポンス生成・描画
```

---

## 2. API定義

### GET /api/products

**クエリパラメータ**

| パラメータ | 型 | 必須 | デフォルト | 説明 |
|------------|----|------|-----------|------|
| category | integer | 任意 | なし | カテゴリID |
| keyword | string | 任意 | なし | 検索キーワード |
| sort | string | 任意 | `new` | `new` / `price_asc` / `price_desc` / `popular` |
| page | integer | 任意 | 1 | ページ番号 |

**レスポンス (200 OK)**

```json
{
  "total": 120,
  "page": 1,
  "per_page": 20,
  "products": [
    {
      "id": 1,
      "name": "サンプル商品A",
      "price": 1980,
      "thumbnail_url": "/images/products/1.jpg",
      "stock": 5,
      "is_available": true
    }
  ]
}
```

## 3. DB アクセス詳細

### テーブル: `products`

| カラム | 型 | 説明 |
|--------|----|------|
| id | INT PK | 商品ID |
| category_id | INT FK | カテゴリID |
| name | VARCHAR(200) | 商品名 |
| description | TEXT | 商品説明 |
| price | INT | 販売価格（税抜） |
| stock | INT | 在庫数 |
| is_published | TINYINT(1) | 公開フラグ |
| created_at | DATETIME | 登録日時 |
| sales_count | INT | 累計販売数（人気順ソート用） |

### 発行SQL（例：キーワード＋カテゴリ絞り込み、新着順）

```sql
SELECT id, name, price, thumbnail_url, stock
FROM products
WHERE is_published = 1
  AND category_id = :category
  AND (name LIKE :keyword OR description LIKE :keyword)
ORDER BY created_at DESC
LIMIT 20 OFFSET :offset;
```

---

## 4. 並び替え条件マッピング

| sort パラメータ | ORDER BY句 |
|----------------|-----------|
| `new` | `created_at DESC` |
| `price_asc` | `price ASC` |
| `price_desc` | `price DESC` |
| `popular` | `sales_count DESC` |

---

## 5. エラーハンドリング

| エラー | 対応 |
|--------|------|
| DB接続エラー | 500エラー画面表示、ログ出力 |
| pageが不正値 | page=1 として処理続行 |
| keywordが101文字以上 | バリデーションエラーメッセージ表示 |

---

## 6. パフォーマンス考慮

- `products(is_published, category_id, created_at)` に複合インデックスを作成
- サムネイル画像はCDN配信（オリジンサーバー負荷軽減）
- 検索結果はRedisに60秒キャッシュ（同一パラメータの場合）
