| 項目 | 内容 |
|------|------|
| 画面ID | SCR-001 |
| 画面名 | 商品一覧 |

| バージョン | 1.0 |

---

## 1. 処理フロー

```
[ページ表示]
    │
    ├─ リクエストパラメータ取得（category, keyword, sort, page）
    │
    ├─ 入力バリデーション
    │       keyword: 最大100文字（超過時 400 エラー）
    │       page: 数値・最小1（不正値は page=1 として処理）
    │       sort: new/price_asc/price_desc/popular のいずれか
    │             （不正値は new として処理）
    │       category: 数値（不正値・存在しないIDは全カテゴリ扱い）
    │
    ├─ Redisキャッシュ確認（キー: products:list:{hash(params)}）
    │       ヒット → キャッシュ返却
    │       ミス  → DBクエリ実行
    │             Redis接続失敗 → フォールバック（DB直接取得）
    │
    ├─ 商品マスタ検索（SELECT）
    │       WHERE 公開フラグ = 1
    │         AND（category指定時）カテゴリID = :category
    │         AND（keyword指定時）商品名 LIKE :keyword
    │              OR 商品説明 LIKE :keyword
    │             ※ keyword パラメータは '%keyword%' としてアプリ側で付加
    │             ※ バインド変数（プレースホルダ）方式でSQLインジェクション対策
    │       ORDER BY [並び替え条件]
    │       LIMIT 20 OFFSET (page-1)*20
    │
    ├─ Redisへキャッシュ保存（TTL: 60秒）
    │
    ├─ DBエラー → 500エラー画面（SCR-ERR）へ遷移・ERRORログ出力
    │
    └─ レスポンス生成・描画
```

---

## 2. API定義

### GET /api/products

**認証**: 不要（公開API。ゲスト・会員どちらもアクセス可）

**クエリパラメータ**

| パラメータ | 型 | 必須 | デフォルト | 説明 |
|------------|----|------|-----------|------|
| category | integer | 任意 | なし | カテゴリID |
| keyword | string | 任意 | なし | 検索キーワード（最大100文字） |
| sort | string | 任意 | `new` | `new` / `price_asc` / `price_desc` / `popular` |
| page | integer | 任意 | 1 | ページ番号 |

**レスポンス (200 OK)**

```json
{
  "total": 120,
  "page": 1,
  "per_page": 20,
  "products": [
    {
      "id": 1,
      "name": "サンプル商品A",
      "price": 1980,
      "thumbnail_url": "/images/products/1.jpg",
      "stock": 5,
      "is_available": true
    }
  ]
}
```

**エラーレスポンス**

| ステータス | エラーコード | 説明 |
|-----------|-------------|------|
| 400 | `VALIDATION_ERROR` | keywordが101文字以上 |
| 500 | `INTERNAL_SERVER_ERROR` | DB/サーバーエラー |

```json
// 400
{ "error": "VALIDATION_ERROR", "message": "キーワードは100文字以内で入力してください" }
// 500
{ "error": "INTERNAL_SERVER_ERROR", "message": "システムエラーが発生しました。しばらく後に再度お試しください" }
```

## 3. DB アクセス詳細

### テーブル: `products`

| カラム | 型 | 説明 |
|--------|----|------|
| id | INT PK | 商品ID |
| category_id | INT FK | カテゴリID |
| name | VARCHAR(200) | 商品名 |
| description | TEXT | 商品説明 |
| price | INT | 販売価格（税抜） |
| stock | INT | 在庫数 |
| is_published | TINYINT(1) | 公開フラグ |
| created_at | DATETIME | 登録日時 |
| sales_count | INT | 累計販売数（人気順ソート用） |
| thumbnail_url | VARCHAR(500) | サムネイル画像URL |

### 発行SQL（例：キーワード＋カテゴリ絞り込み、新着順）

```sql
-- keyword パラメータはアプリ側で '%' を前後に付加してバインド
-- 例: :keyword = '%サンプル%'
SELECT id, name, price, thumbnail_url, stock
FROM products
WHERE is_published = 1
  AND category_id = :category
  AND (name LIKE :keyword OR description LIKE :keyword)
ORDER BY created_at DESC
LIMIT 20 OFFSET :offset;
```

**SQLインジェクション対策**: バインド変数（プレースホルダ）方式を採用。`%` はアプリ側でパラメータ値に付加し、バインド変数にセットする。

## 4. 並び替え条件マッピング

| sort パラメータ | ORDER BY句 |
|----------------|-----------|
| `new` | `created_at DESC` |
| `price_asc` | `price ASC` |
| `price_desc` | `price DESC` |
| `popular` | `sales_count DESC` |

## 5. エラーハンドリング

| エラー | 対応 | ログレベル |
|--------|------|-----------|
| DB接続エラー | 500エラー画面（SCR-ERR）表示。DBエラーコード・リクエストパラメータをERRORログ出力 | ERROR |
| DBタイムアウト（3秒） | 500エラー画面表示。タイムアウト発生箇所・パラメータをERRORログ出力 | ERROR |
| pageが不正値 | page=1 として処理続行 | WARN |
| keywordが101文字以上 | 400レスポンス返却・バリデーションエラーメッセージ表示 | INFO |
| sortが不正値 | sort=new として処理続行 | WARN |
| categoryが不正値 | 全カテゴリ扱いとして処理続行 | WARN |
| Redisエラー | フォールバック（DB直接取得）して処理続行。WARNログ出力 | WARN |
| Redisタイムアウト（500ms） | フォールバック（DB直接取得）して処理続行 | WARN |

**エラーログ出力項目**: タイムスタンプ、ログレベル、エラーコード、リクエストURL/パラメータ、スタックトレース（500エラー時のみ）

**エラーレスポンスの内部情報**: 500エラー画面にはスタックトレース・DB内部情報を表示しない。エラーはログに限定して記録する。

**リトライ**: DB接続エラーはリトライなし（上位にエラーを返却）。Redisエラーはフォールバックで継続。

## 6. パフォーマンス考慮

- `products(is_published, category_id, created_at)` に複合インデックスを作成
- サムネイル画像はCDN配信（オリジンサーバー負荷軽減）
- 検索結果はRedisに60秒キャッシュ（キーはパラメータのハッシュ値: `products:list:{md5(category,keyword,sort,page)}`）
- 商品情報更新時はキャッシュをパージ（キープレフィックス `products:list:*` を削除）
- Redisダウン時はDBから直接取得するフォールバックを実装
- OFFSETページネーションを採用。ページ番号上限500（10,000件上限）を設け、大ページ番号での性能劣化を抑制
- DBタイムアウト: 接続タイムアウト3秒、クエリタイムアウト5秒
