## 1. 概要

注文確定後に表示する完了通知画面。注文番号・合計金額・お届け先を表示する。
コンビニ払い選択時は支払い番号も表示する。

**対象ユーザー**: ログイン済みユーザーのみ。他ユーザーのorder_idへのアクセスは403を返す。

**レスポンシブ対応**: スマートフォン・タブレット対応必須。

## 3. 機能一覧

| No | 機能名 | 説明 |
|----|--------|------|
| F-01 | 注文完了表示 | 注文番号・注文内容・配送先・決済情報を表示する |
| F-02 | 注文確認メール送信 | 注文完了と同時に登録メールアドレスへ確認メールを非同期送信する |
| F-03 | トップへ戻るリンク | 商品一覧画面（SCR-001）へ遷移する |
| F-04 | 無効アクセス処理 | order_idなし/不正なorder_idへのアクセスは商品一覧へリダイレクト |
| F-05 | メール送信失敗通知 | 送信失敗時は画面上に「メール送信に失敗しました。ご注文は完了しています。」と表示する |
| F-06 | コンビニ払い期限切れ表示 | 支払い期限を過ぎた場合、その旨を表示しカスタマーサポートへ誘導する |

## 4. 画面遷移

| 操作 | 遷移先 |
|------|--------|
| 「トップページへ戻る」リンク | 商品一覧画面（SCR-001） |
| order_idなしでの直接アクセス | 商品一覧画面（SCR-001）へリダイレクト |
| 他ユーザーのorder_idへのアクセス | エラー画面（SCR-ERR、403） |
| ブラウザバック | PRGパターンにより注文完了画面に留まる（二重注文防止） |

## 5. 業務ルール

- 直接URL入力（order_id なし）でのアクセスは商品一覧へリダイレクト
- 同一 order_id へのリロード・再アクセスは正常表示する（冪等性確保）
- 注文確認メールはバックグラウンドで非同期送信する
- コンビニ払いのお支払い期限は注文日から3日後（72時間）
- 注文番号は最大10桁の数字とする
- 支払い番号は12桁の数字とする
- 合計金額は最大10桁（消費税込み、上限9,999,999,999円）
- 他ユーザーのorder_idへのアクセスは403を返し詳細情報を表示しない
