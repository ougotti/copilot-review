# 詳細設計書 – 注文完了画面

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-005 |
| 画面名 | 注文完了 |
| 作成日 | 2026-04-08 |
| バージョン | 1.0 |

---

## 1. 処理フロー

```
[ページ表示]
    │
    ├─ パスパラメータから order_id 取得（integer, 必須）
    │       なし / 不正値 → SCR-001 へリダイレクト
    │
    ├─ 認証チェック: セッションCookie必須（未ログイン時はSCR-LOGINへ）
    │
    ├─ orders JOIN order_items で注文情報取得
    │       存在しない → 404
    │       他ユーザーの注文 → 403
    │       DBエラー → 500エラー画面（SCR-ERR）・ERRORログ出力
    │
    ├─ Cache-Control: no-store ヘッダーを付与（注文情報のキャッシュ禁止）
    │
    └─ 画面描画

[メール送信（非同期キュー）]
    │
    ├─ 注文確定時にメールキューへ enqueue（注文トランザクション完了後）
    │       enqueue失敗時もトランザクションには影響しない
    │
    ├─ ワーカーがキューを処理
    │       宛先: ユーザー登録メールアドレス
    │       件名: 【ECサイト】ご注文ありがとうございます（#ORDER_ID）
    │
    └─ 送信失敗時: 最大3回リトライ（指数バックオフ）
            3回失敗後: 画面上にメール送信失敗メッセージを表示
```

---

## 2. API定義

### GET /api/orders/{id}

**認証**: セッションCookie必須

**パスパラメータ**: `id` – integer, 必須

**レスポンス (200 OK)**

```json
{
  "order_id": 10001,
  "total_amount": 4356,
  "payment_method": "credit_card",
  "payment_status": "paid",
  "convenience_payment_number": null,
  "convenience_due_date": null,
  "shipping": {
    "name": "山田 太郎",
    "address": "東京都千代田区千代田1-1"
  },
  "items": [
    {
      "product_name": "サンプル商品A",
      "unit_price": 2178,
      "quantity": 2,
      "subtotal": 4356
    }
  ],
  "created_at": "2026-04-08T14:05:27Z"
}
```

**エラーレスポンス**

| ステータス | エラーコード | 説明 |
|-----------|-------------|------|
| 404 | `NOT_FOUND` | 注文が存在しない |
| 403 | `FORBIDDEN` | 他ユーザーの注文へのアクセス |
| 500 | `INTERNAL_SERVER_ERROR` | DBエラー |

```json
// 404
{ "error": "NOT_FOUND", "message": "注文が見つかりません" }
// 403
{ "error": "FORBIDDEN", "message": "アクセス権限がありません" }
// 500
{ "error": "INTERNAL_SERVER_ERROR", "message": "システムエラーが発生しました" }
```

**注意**: 403レスポンスに注文の存在有無を示す情報を含めないこと（情報漏洩防止）。

## 4. エラーハンドリング

| エラー | 対応 | ログレベル | ログ内容 |
|--------|------|-----------|----------|
| order_idが存在しない | 404ページ表示 | INFO | リクエストのorder_id |
| 他ユーザーの注文へのアクセス | 403レスポンス（詳細非開示） | WARN | アクセス者セッションID・リクエストorder_id |
| セッション切れ | ログイン画面（SCR-LOGIN）へリダイレクト | INFO | |
| DBエラー | 500エラー画面（SCR-ERR）表示 | ERROR | リクエストID・SQLエラーコード |
| メール送信失敗 | リトライキューに積む（画面表示はブロックしない）。3回失敗後は画面上に通知 | WARN | 注文ID・送信先メールアドレス・エラーコード |

**タイムアウト**: DBクエリタイムアウト3秒。

**XSS対策**: メールテンプレートはテンプレートエンジンのエスケープ機能を使用してHTMLタグをエスケープする。

**SQLインジェクション対策**: バインド変数（プレースホルダ）方式を採用。

---

## 5. セキュリティ考慮

- order_id はランダムな UUID を併用し、連番推測による他ユーザーの注文閲覧を防止する
- セッションIDと order の session_id が一致しない場合は 403 を返す
