| 作成日 | 2026-04-08 |
| バージョン | 1.0 |

---

## 1. 処理フロー

```
[ページ表示]
    │
    ├─ クエリパラメータから order_id 取得
    │       なし → SCR-001 へリダイレクト
    │
    ├─ orders JOIN order_items で注文情報取得
    │       存在しない → 404
    │
    └─ 画面描画

[メール送信（非同期キュー）]
    │
    ├─ 注文確定時にメールキューへ enqueue
    │
    ├─ ワーカーがキューを処理
    │       宛先: ユーザー登録メールアドレス
    │       件名: 【ECサイト】ご注文ありがとうございます（#ORDER_ID）
    │
    └─ 送信失敗時: 最大3回リトライ（指数バックオフ）
```

  {{shipping_name}}

---

## 4. エラーハンドリング

| エラー | 対応 |
|--------|------|
| order_id が存在しない | 404ページ表示 |
| メール送信失敗 | リトライキューに積む（画面表示はブロックしない） |

---

## 5. セキュリティ考慮

- order_id はランダムな UUID を併用し、連番推測による他ユーザーの注文閲覧を防止する
- セッションIDと order の session_id が一致しない場合は 403 を返す
