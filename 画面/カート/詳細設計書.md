## 1. 処理フロー

```
[ページ表示]
    │
    ├─ セッションIDを取得
    │
    ├─ cart_items JOIN products で商品情報を取得
    │
    ├─ 在庫状況を最新に同期
    │       在庫 < カート数量の場合 → カート数量を在庫数に修正し警告表示
    │
    └─ 合計金額を算出して描画

[数量変更]
    │
    ├─ PATCH /api/cart/items/{id}  { quantity: N }
    │
    ├─ quantity = 0 の場合、DELETE 処理へ
    │
    ├─ 在庫チェック
    │
    ├─ cart_items を UPDATE
    │
    └─ 新しい合計金額をレスポンスに含めてフロントを更新

[商品削除]
    │
    ├─ DELETE /api/cart/items/{id}
    │
    └─ cart_items から該当レコードを DELETE
```

## 2. API定義

### GET /api/cart

**認証**: セッションCookie必須

**レスポンス (200 OK)**

```json
{
  "item_count": 2,
  "items": [
    {
      "cart_item_id": 42,
      "product_id": 1,
      "name": "サンプル商品A",
      "thumbnail_url": "/images/1.jpg",
      "unit_price": 2178,
      "quantity": 2,
      "subtotal": 4356,
      "stock": 5
    }
  ],
  "total": 4356
}
```

**エラーレスポンス**

| ステータス | エラーコード | 説明 |
|-----------|-------------|------|
| 500 | `INTERNAL_SERVER_ERROR` | DBエラー |

### PATCH /api/cart/items/{id}

**認証**: セッションCookie必須

**CSRFトークン**: リクエストヘッダーに `X-CSRF-Token` を含めること

**リクエストティールド**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| quantity | integer | ○ | 数量（最小:1） |

```json
// Request
{ "quantity": 3 }

// Response 200
{ "subtotal": 6534, "cart_total": 6534 }
```

**エラーレスポンス**

| ステータス | エラーコード | 説明 |
|-----------|-------------|------|
| 400 | `VALIDATION_ERROR` | quantityが1未満 |
| 404 | `NOT_FOUND` | アイテムが存在しない |
| 409 | `OUT_OF_STOCK` | 在庫不足 |
| 500 | `INTERNAL_SERVER_ERROR` | DBエラー |

```json
// 404
{ "error": "NOT_FOUND", "message": "カートアイテムが見つかりません" }
// 409
{ "error": "OUT_OF_STOCK", "message": "在庫が不足しています" }
// 500
{ "error": "INTERNAL_SERVER_ERROR", "message": "システムエラーが発生しました" }
```

### DELETE /api/cart/items/{id}

**認証**: セッションCookie必須

**CSRFトークン**: リクエストヘッダーに `X-CSRF-Token` を含めること

```json
// Response 200
{ "message": "削除しました", "cart_total": 0 }
```

**エラーレスポンス**

| ステータス | エラーコード | 説明 |
|-----------|-------------|------|
| 404 | `NOT_FOUND` | アイテムが存在しない |
| 500 | `INTERNAL_SERVER_ERROR` | DBエラー |

## 3. DB アクセス詳細

### テーブル: `cart_items`（インデックス追記）

- `(session_id, product_id)` UNIQUE制約およびインデックス（カート一覧取得の高速化）

### カート一覧取得 SQL

```sql
SELECT
  ci.id         AS cart_item_id,
  p.id          AS product_id,
  p.name,
  p.thumbnail_url,
  FLOOR(p.price * 1.10) AS unit_price,
  ci.quantity,
  FLOOR(p.price * 1.10) * ci.quantity AS subtotal,
  p.stock
FROM cart_items ci
JOIN products p ON ci.product_id = p.id
WHERE ci.session_id = :session_id
ORDER BY ci.created_at ASC;
```

### 商品削除 SQL

```sql
DELETE FROM cart_items
WHERE id = :cart_item_id AND session_id = :session_id;
```

**注意**: DECIMAL型で税込計算を行う（FLOAT型は浮動小数点誤差が発生するため非推奨）

**キャッシュ方針**: カート情報（価格・在庫）はvolatileなデータのためキャッシュ不可。常に最新値をDBから取得する。

### 数量更新 SQL

```sql
UPDATE cart_items
SET quantity = :quantity, updated_at = NOW()
WHERE id = :cart_item_id AND session_id = :session_id;
```

## 4. エラーハンドリング

| エラー | 対応 | ログレベル | ログ内容 |
|--------|------|-----------|----------|
| 在庫不足（数量変更時） | 409レスポンス「在庫が不足しています」 | INFO | product_id・要求数・在庫数 |
| カートアイテムが存在しない | 404レスポンス | INFO | リクエストのcart_item_id・セッションID |
| DBエラー | 500エラー画面（SCR-ERR）表示 | ERROR | リクエストID・セッションID・SQLエラーコード |
| CSRFトークン不一致 | 403レスポンス | WARN | リクエストIP・セッションID |

**リトライ**: ブラウザリトライなし（エラーメッセージ表示のみ）。DB接続エラーもリトライなしとし500エラー画面へ遷移。

**タイムアウト**: DB接続タイムアウト1秒、クエリタイムアウト3秒。
