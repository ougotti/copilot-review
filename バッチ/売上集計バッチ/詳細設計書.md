```

    │
    ├─ 環境変数読み込み（DB接続、出力ディレクトリ等）
    │       読み込み失敗 → ERRORログ出力、終了コード2で終了

    │

    │       ├─ 日次集計 SELECT（orders + order_items）
    │       │       DBエラー → rollback → ERRORログ出力、終了コード2で終了
    │       ├─ sales_daily テーブルへ UPSERT
    │       ├─ 商品別集計 SELECT（order_items + products）
    │       └─ sales_daily_products テーブルへ UPSERT
    ├─ [トランザクションコミット]
    │       失敗 → rollback → ERRORログ出力、終了コード9で終了
    │
    ├─ CSVファイル生成（${REPORT_OUTPUT_DIR}/sales_YYYYMMDD.csv）
    │       ├─ 一時ファイル（.tmp）に書き込み後にリネーム（ディスク障害対応）
    │       └─ 失敗 → ERRORログ出力、終了コード1で終了
    │
    ├─ ログ「集計完了: 売上 ¥XXX / X件」記録
    │
    └─ 終了（正常、終了コード0）
```

## 3. 集計 SQL

### 日次サマリー集計

```sql
-- DATE(o.created_at) = :target_date はインデックスを使えないため範囲条件を使用
SELECT
  SUM(o.total_amount)       AS total_amount,
  COUNT(o.id)               AS order_count
FROM orders o
WHERE o.payment_status = 'paid'
  AND o.created_at >= :target_start
  AND o.created_at < :target_end;
-- :target_start = '2026-04-07 00:00:00' (JST)
-- :target_end   = '2026-04-08 00:00:00' (JST)
```

**`created_at` へのインデックス**: `orders(payment_status, created_at)` に複合インデックスを作成することを推奨。

### 商品別集計

```sql
SELECT
  oi.product_id,
  oi.product_name,
  p.category_id,
  SUM(oi.quantity)          AS quantity_sold,
  SUM(oi.unit_price * oi.quantity) AS total_amount
FROM order_items oi
JOIN orders o ON oi.order_id = o.id
JOIN products p ON oi.product_id = p.id
WHERE o.payment_status = 'paid'
  AND o.created_at >= :target_start
  AND o.created_at < :target_end
GROUP BY oi.product_id, oi.product_name, p.category_id;
```

### sales_daily UPSERT

```sql
INSERT INTO sales_daily (sales_date, total_amount, order_count, created_at, updated_at)
VALUES (:sales_date, :total_amount, :order_count, NOW(), NOW())
ON DUPLICATE KEY UPDATE
  total_amount = :total_amount,
  order_count  = :order_count,
  updated_at   = NOW();
```

## 6. エラーハンドリング

| エラー | 処理継続/中断 | 対応 | 終了コード |
|--------|------------|------|-----------|
| 環境変数読み込み失敗 | 中断 | ERRORログ出力 | 2 |
| ログファイルオープン失敗 | 中断 | 標準エラー出力 | 9 |
| DB接続エラー | 中断 | ERRORログ出力 | 2 |
| DBクエリエラー | 中断 | rollback・ERRORログ出力 | 2 |
| トランザクションコミット失敗 | 中断 | rollback・ERRORログ出力 | 9 |
| CSV書き込み失敗 | 中断（DB更新は済み） | ERRORログ出力 | 1 |

**タイムアウト**: DBクエリタイムアウト60秒（大量データ集計に対応）。

**リトライ**: DBエラーはリトライなし。

**監視連携**: 終了コードを監視ツールへ連携。異常終了時はアラートをメール/Slackで通知する。

**パフォーマンス**: 対象注文が10万件を超える場合は月次バッチへの移行を検討する。想定処理時間10分以内。

**CSVディスク障害対応**: 一時ファイル（`.tmp`）に書き込み後、完了したらリネームする。中断時に不完全なCSVが残らないようにする。

- UPSERT により同一日付での再実行は安全（冪等性確保）
