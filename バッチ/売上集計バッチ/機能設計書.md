# 機能設計書 – 売上集計バッチ

| 項目 | 内容 |
|------|------|
| バッチID | BAT-002 |
| バッチ名 | 売上集計バッチ |
| 作成日 | 2026-04-08 |
| バージョン | 1.0 |

---

## 1. 概要

前日の注文データを集計し、日次売上レポートを `sales_daily` テーブルへ格納するバッチ処理。
経営ダッシュボードやCSVエクスポートのデータソースとして利用される。

---

## 2. 実行スケジュール

| 項目 | 内容 |
|------|------|
| 実行タイミング | 毎日 03:00（JST） |
| 実行方式 | cron / タスクスケジューラ |
| 想定処理時間 | 10分以内 |

---

## 3. 処理概要

1. 対象日（前日）の `orders`・`order_items` から売上データを集計
2. 集計結果を `sales_daily` テーブルへ UPSERT
3. 日次売上サマリーCSVを所定ディレクトリへ出力
4. 処理結果をログファイルへ出力

---

## 4. 集計内容

| 集計項目 | 説明 |
|----------|------|
| 売上合計（税込） | 決済済み注文の合計金額 |
| 注文件数 | 決済済み注文の件数 |
| 商品別売上数 | 商品ごとの販売数量合計 |
| カテゴリ別売上 | カテゴリごとの売上金額合計 |

---

## 5. 入力

| 種別 | 内容 |
|------|------|
| DBテーブル | `orders`、`order_items`、`products` |
| 引数（省略可） | 集計対象日 `--date YYYY-MM-DD`（省略時は前日） |

---

## 6. 出力

| 種別 | 内容 |
|------|------|
| DBテーブル | `sales_daily`（日次集計INSERT/UPDATE） |
| CSVファイル | `reports/sales_YYYYMMDD.csv` |
| ログ | `logs/bat002_YYYYMMDD.log` |

---

## 7. 業務ルール

- 集計対象は `payment_status = 'paid'` の注文のみ
- 同一日付のデータが既存の場合は上書き（UPSERT）する（再実行安全性の確保）
- キャンセル注文（将来実装）は集計対象外とする
- 集計対象日の注文が0件の場合も0件レコードを挿入してログに記録する
