## 1. 処理フロー

```
[起動]
    │
    ├─ 設定ファイル読み込み（メールアドレス、閾値等）
    │
    ├─ ログファイルオープン（logs/bat001_YYYYMMDD.log）
    │
    ├─ 実行日を取得（`run_date = TODAY()`）
    │
    ├─ 補充対象商品を取得（SELECT）
    │       is_published = 1
    │       AND stock <= reorder_point
    │       AND id NOT IN (
    │           SELECT product_id FROM stock_alerts
    │           WHERE DATE(alerted_at) = run_date
    │       )
    │
    ├─ 対象商品が0件
    │       → ログ「対象商品なし」記録 → 終了（正常）
    │
    ├─ メール本文生成（商品一覧を表形式で記載）
    │
    ├─ メール送信
    │       失敗 → エラーログ出力、終了コード 1 で終了
    │
    ├─ stock_alerts テーブルへ INSERT（対象商品分）
    │
    ├─ ログ「処理完了: X件」記録
    │
    └─ 終了（正常、終了コード 0）
```

## 2. DB アクセス詳細

### テーブル: `products`（追加カラム）

| カラム | 型 | 説明 |
|--------|----|------|
| reorder_point | INT | 補充閾値（デフォルト: 10） |

### テーブル: `stock_alerts`

| カラム | 型 | 説明 |
|--------|----|------|
| id | INT PK | |
| product_id | INT FK | |
| stock_at_alert | INT | アラート時在庫数 |
| is_urgent | TINYINT(1) | 在庫0フラグ |
| alerted_at | DATETIME | アラート日時 |

### 補充対象商品取得 SQL

```sql
SELECT p.id, p.name, p.stock, p.reorder_point,
       CASE WHEN p.stock = 0 THEN 1 ELSE 0 END AS is_urgent
FROM products p
WHERE p.is_published = 1
  AND p.stock <= p.reorder_point
  AND p.id NOT IN (
    SELECT product_id FROM stock_alerts
    WHERE DATE(alerted_at) = CURDATE()
  )
ORDER BY p.stock ASC;
```

### アラート履歴 INSERT SQL

```sql
INSERT INTO stock_alerts (product_id, stock_at_alert, is_urgent, alerted_at)
VALUES (:product_id, :stock, :is_urgent, NOW());
```

## 3. メール仕様

| 項目 | 内容 |
|------|------|
| 件名（通常） | 【在庫補充アラート】YYYY/MM/DD 補充対象: X件 |
| 件名（緊急含む） | [緊急]【在庫補充アラート】YYYY/MM/DD 補充対象: X件 |
| 宛先 | 設定ファイルの `ALERT_MAIL_TO` |

**本文例**

```
以下の商品が在庫補充閾値を下回りました。

商品ID | 商品名             | 現在庫 | 閾値 | 状況
-------+--------------------+--------+------+------
     1 | サンプル商品A      |      3 |   10 | 要補充
     2 | サンプル商品B      |      0 |    5 | ★緊急
```

## 4. 設定ファイル（`.env` / 環境変数）

| キー | 説明 | デフォルト |
|------|------|-----------|
| `ALERT_MAIL_TO` | アラート送信先メールアドレス | 必須 |
| `MAIL_FROM` | 送信元メールアドレス | `no-reply@example.com` |
| `DEFAULT_REORDER_POINT` | 補充閾値デフォルト | `10` |

## 5. ログ仕様

- フォーマット: `[YYYY-MM-DD HH:mm:ss] [LEVEL] メッセージ`
- 保存先: `logs/bat001_YYYYMMDD.log`
- ローテーション: 30日保持
