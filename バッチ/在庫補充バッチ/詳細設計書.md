---

## 1. 処理フロー

```
[起動]
    │
    ├─ 環境変数読み込み（DB接続、SMTP、送信先メールアドレス等）
    │       読み込み失敗 → ERRORログ出力、終了コード2で終了
    │
    ├─ ログファイルオープン（logs/bat001_YYYYMMDD.log、UTF-8, LF）
    │       オープン失敗 → 標準エラー出力へ書き込み、終了コード9で終了
    │
    ├─ 実行日を取得（`run_date = TODAY()`）
    │
    ├─ 補充対象商品を取得（SELECT）
    │       is_published = 1
    │       AND stock <= reorder_point
    │       AND id NOT IN (
    │           SELECT product_id FROM stock_alerts
    │           WHERE alerted_at >= :start AND alerted_at < :end  ← インデックス活用
    │       )
    │       DBエラー → ERRORログ出力、終了コード2で終了
    │
    ├─ 対象商品が0件
    │       → ログ「対象商品なし」記録 → 終了（正常、終了コード0）
    │
    ├─ メール本文生成（商品一覧を表形式で記載）
    │
    ├─ [トランザクション開始]
    │       └─ stock_alerts テーブルへ INSERT（対象商品分）
    ├─ [トランザクションコミット]
    │       INSERT失敗 → rollback → ERRORログ出力、終了コード9で終了
    │
    ├─ メール送信（最大3回リトライ、指数バックオフ）
    │       3回失敗 → WARNログ出力、終了コード1で終了
    │
    ├─ ログ「処理完了: X件」記録
    │
    └─ 終了（正常、終了コード0）
```

---

## 2. DB アクセス詳細

### テーブル: `products`（追加カラム）

| カラム | 型 | 説明 |
|--------|----|------|
| reorder_point | INT | 補充閾値（デフォルト: 10） |

### テーブル: `stock_alerts`

| カラム | 型 | 説明 |
|--------|----|------|
| id | INT PK | |
| product_id | INT FK | |
| stock_at_alert | INT | アラート時在庫数 |
| is_urgent | TINYINT(1) | 在庫0フラグ |
| alerted_at | DATETIME | アラート日時 |

### 補充対象商品取得 SQL

```sql
-- alerted_at に範囲条件を使用しインデックスを活用する
-- DATE(alerted_at) = CURDATE() はインデックスを使えないため不使用
SELECT p.id, p.name, p.stock, p.reorder_point,
       CASE WHEN p.stock = 0 THEN 1 ELSE 0 END AS is_urgent
FROM products p
WHERE p.is_published = 1
  AND p.stock <= p.reorder_point
  AND p.id NOT IN (
    SELECT product_id FROM stock_alerts
    WHERE alerted_at >= :today_start AND alerted_at < :tomorrow_start
  )
ORDER BY p.stock ASC;
```

### アラート履歴 INSERT SQL

```sql
INSERT INTO stock_alerts (product_id, stock_at_alert, is_urgent, alerted_at)
VALUES (:product_id, :stock, :is_urgent, NOW());
```

## 3. メール仕様

| 項目 | 内容 |
|------|------|
| 件名（通常） | 【在庫補充アラート】YYYY/MM/DD 補充対象: X件 |
| 件名（緊急含む） | [緊急]【在庫補充アラート】YYYY/MM/DD 補充対象: X件 |
| 宛先 | 設定ファイルの `ALERT_MAIL_TO` |

**本文例**

```
以下の商品が在庫補充閾値を下回りました。

商品ID | 商品名             | 現在庫 | 閾値 | 状況
-------+--------------------+--------+------+------
     1 | サンプル商品A      |      3 |   10 | 要補充
     2 | サンプル商品B      |      0 |    5 | ★緊急
```

## 4. 設定ファイル（環境変数）

| キー | 説明 | デフォルト |
|------|------|-----------|
| `DB_HOST` | DBホスト名 | 必須 |
| `DB_PORT` | DBポート | 3306 |
| `DB_NAME` | DB名 | 必須 |
| `DB_USER` | DBユーザー | 必須 |
| `DB_PASS` | DBパスワード（シークレット管理ツールから注入） | 必須 |
| `SMTP_HOST` | SMTPホスト | 必須 |
| `SMTP_PORT` | SMTPポート | 587 |
| `SMTP_USER` | SMTPユーザー | 必須 |
| `SMTP_PASS` | SMTPパスワード（シークレット管理ツールから注入） | 必須 |
| `ALERT_MAIL_TO` | アラート送信先メールアドレス | 必須 |

## 5. ログ仕様

- フォーマット: `[YYYY-MM-DD HH:mm:ss] [LEVEL] メッセージ`
- 保存先: `logs/bat001_YYYYMMDD.log`
- 文字エンコーディング: UTF-8, LF
- ローテーション: 30日保持
- ログレベル定義: ERROR（処理継続不可）/ WARN（処理続行可）/ INFO（正常系情報）

---

## 6. エラーハンドリング

| エラー | 処理継続/中断 | 対応 | 終了コード |
|--------|------------|------|-----------|
| 設定読み込み失敗 | 中断 | ERRORログ出力 | 2 |
| ログファイルオープン失敗 | 中断 | 標準エラー出力 | 9 |
| DB接続エラー | 中断 | ERRORログ出力 | 2 |
| DBクエリエラー | 中断 | ERRORログ出力（SQLエラーコード含む） | 2 |
| stock_alerts INSERT失敗 | 中断 | rollback・ERRORログ出力 | 9 |
| メール送信失敗 | 継続（リトライ後） | WARNログ出力（3回失敗で終了コード1） | 1 |

**タイムアウト**: DB接続タイムアウト30秒、メール送信タイムアウト10秒。

**リトライ**: メール送信は最大3回、指数バックオフ（1秒→2秒→4秒）。DBエラーはリトライなし。

**監視連携**: 終了コードを監視ツール（Zabbix/Datadog）へ連携。異常終了時はアラートを通知する。

**パフォーマンス**: 対象商品数が最大1,000件を超える場合はチャンク処理（500件単位）を行う。想定処理時間は5分以内。
