# 機能設計書 – 在庫補充バッチ

| 項目 | 内容 |
|------|------|
| バッチID | BAT-001 |
| バッチ名 | 在庫補充バッチ |
| 作成日 | 2026-04-08 |
| バージョン | 1.0 |

---

## 1. 概要

在庫数が閾値を下回った商品を検出し、仕入れ担当者へアラートメールを送信する夜間バッチ処理。
将来的には外部発注システムへの自動発注も想定する。

---

## 2. 実行スケジュール

| 項目 | 内容 |
|------|------|
| 実行タイミング | 毎日 02:00（JST） |
| 実行方式 | cron / タスクスケジューラ |
| 想定処理時間 | 5分以内 |

---

## 3. 処理概要

1. `products` テーブルから在庫補充閾値（`reorder_point`）以下の公開商品を取得
2. 対象商品リストを在庫補充アラートメールとして担当者メールアドレスへ送信
3. `stock_alerts` テーブルへアラート送信履歴を記録
4. 処理結果をログファイルへ出力

---

## 4. 入力

| 種別 | 内容 |
|------|------|
| DBテーブル | `products`（在庫数・補充閾値） |
| 設定ファイル | アラート送信先メールアドレス、閾値デフォルト値 |

---

## 5. 出力

| 種別 | 内容 |
|------|------|
| メール | 在庫補充アラートメール（担当者宛） |
| DBテーブル | `stock_alerts`（アラート履歴INSERT） |
| ログ | `logs/bat001_YYYYMMDD.log` |

---

## 6. 業務ルール

- 同一商品に対して同日内に複数回アラートを送信しない（`stock_alerts` で重複チェック）
- 在庫数が0の商品は「緊急」フラグを付けてメール件名に `[緊急]` プレフィックスを追加
- 公開フラグが「非公開」の商品はアラート対象外
- 処理対象商品が0件の場合はメール送信をスキップし、ログに「対象商品なし」と記録
