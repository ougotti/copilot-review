## 1. 概要

在庫数が閾値を下回った商品を検出し、仕入れ担当者へアラートメールを送信する夜間バッチ処理。
将来的には外部発注システムへの自動発注も想定する。

---

## 2. 実行スケジュール

| 項目 | 内容 |
|------|------|
| 実行タイミング | 毎日 02:00（JST） |
| cron式 | `0 2 * * *` |
| 実行方式 | cron / タスクスケジューラ |
| 想定処理時間 | 5分以内 |
| 依存関係 | 他バッチとの依存なし（独立実行可能） |
| スケジュール障害対応 | 実行失敗時は監視ツール（Zabbix/Datadog）からアラートを担当者メールへ通知。手動再実行手順はRunbook参照 |
| 冪等性 | 同日に再実行しても `stock_alerts` の重複チェックにより二重アラートは発生しない |

## 2.1 起動引数・環境変数

| 変数名 | 説明 | 必須 |
|--------|------|------|
| `DB_HOST` | データベースホスト名 | ○ |
| `DB_PORT` | データベースポート | ○ |
| `DB_NAME` | データベース名 | ○ |
| `DB_USER` | データベースユーザー | ○ |
| `DB_PASSWORD` | データベースパスワード（シークレット管理ツールから注入） | ○ |
| `SMTP_HOST` | SMTPサーバーホスト | ○ |
| `SMTP_PORT` | SMTPサーバーポート | ○ |
| `SMTP_USER` | SMTPユーザー | ○ |
| `SMTP_PASS` | SMTPパスワード（シークレット管理ツールから注入） | ○ |
| `ALERT_MAIL_TO` | アラート送信先メールアドレス | ○ |

**セキュリティ**: パスワード類はプレーンテキストで設定ファイルに記載せず、シークレット管理ツール（AWS Secrets Manager等）から環境変数として注入する。

---

## 3. 処理概要

1. `products` テーブルから在庫補充閾値（`reorder_point`）以下の公開商品を取得
2. 対象商品リストを在庫補充アラートメールとして担当者メールアドレスへ送信
3. `stock_alerts` テーブルへアラート送信履歴を記録
4. 処理結果をログファイルへ出力

---

## 4. 入力

| 種別 | 内容 |
|------|------|
| DBテーブル | `products`（在庫数・補充閾値） |
| DBテーブル | `stock_alerts`（重複チェック用） |
| 環境変数 | DB接続情報、SMTP設定、送信先メールアドレス（§2.1参照） |

---

## 5. 出力

| 種別 | 内容 |
|------|------|
| メール | 在庫補充アラートメール（担当者宛） |
| DBテーブル | `stock_alerts`（アラート履歴INSERT） |
| ログ | `logs/bat001_YYYYMMDD.log`（UTF-8, LF） |

---

## 6. 業務ルール

- 同一商品に対して同日内に複数回アラートを送信しない（`stock_alerts` で重複チェック）
- 在庫数が0の商品は「緊急」フラグを付けてメール件名に `[緊急]` プレフィックスを追加
- 公開フラグが「非公開」の商品はアラート対象外
- 処理対象商品が0件の場合はメール送信をスキップし、ログに「対象商品なし」と記録
